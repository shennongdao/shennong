# 神农书库

神农书库（ShenNong Library）是一个去中心化且开源的小说评价平台，目标是让读者更容易找到适合自己的书，并让评分、标签、贡献记录和治理数据尽量长期可用、可恢复、可迁移。

项目使用 ARNS 名称 `shennong` 作为统一入口，使用 AO 协议作为权威状态层，使用 Arweave 保存可长期访问的数据，使用网关提供索引、缓存和查询加速。

## 当前状态

仓库当前处于架构设计和第一阶段最小闭环规格确认阶段。

第一阶段不优先实现小说入库、标签、评分、论坛等完整产品功能，而是先跑通开发贡献闭环：

- 创建或连接钱包；
- 创建 SNID；
- 初始化开发 DAO、推广 DAO、内容 DAO；
- 创建、领取、提交、挑战和裁决任务；
- 发放 `tSNLT`；
- 记录不可转让声誉；
- 支持用户间 `tSNLT` 自由转账；
- 追踪 `tSNLT` 转正式 `SNLT` 的转换资格；
- 通过网关索引身份、DAO、任务、付款、转账、裁决和声誉。

## 核心约定

- ARNS 名称：所有阶段统一使用 `shennong`。
- 测试代币符号：`tSNLT`。
- 正式代币符号：`SNLT`。
- 代币精度：`12`。
- 代币发行：DAO 资金池初始余额为 `0`，按每日增发规则进入资金池。
- 任务付款：所有面向个人的钱包付款必须经过任务模块。
- 声誉：只记录确认后的人类劳动贡献，绑定 SNID，不随代币转账迁移。
- 用户转账：第一阶段支持用户间自由转账，转换资格按来源批次随余额移动。

## 仓库结构

```text
shennong/
  apps/

    gateway/    网关 API、索引服务和健康检查
    crawler/    数据提供商爬虫应用
  protocols/
    ao/         AO 协议进程
  packages/
    sdk/        前端、网关、爬虫和外部集成复用的 SDK
    types/      共享类型
    config/     共享配置
```

## 本地开发

当前仓库尚未引入统一的包管理、构建和启动脚本。实现阶段开始后，应在根目录补充统一开发命令，并在对应模块 README 中补充模块级启动、测试和部署说明。

在添加实现代码前，应先阅读第一阶段设计文档，并保持协议消息、共享类型、网关 API、钱包方案和权限模型一致。

## 贡献方向

第一阶段优先欢迎以下贡献：

- AO 协议进程实现；
- 共享类型和 SDK 实现；
- 网关索引与读模型实现；
- 前端钱包、SNID、DAO、任务、代币、声誉页面实现；
- 任务模板、测试用例和部署说明；
- 文档校对和规则补充。

发生架构改动时，必须同步更新 [架构设计](apps/web/docs/架构设计.md)。
